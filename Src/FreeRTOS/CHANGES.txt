ЭТАП 1: ПАРСИНГ ПАРАМЕТРОВ
═══════════════════════════════════════════════════════════════════════════════

Добавлены 4 helper функции (строки 33-84):
────────────────────────────────────────

  prvGetParamInt(cmd, param_num, min, max)
  • Получить целое число из параметра
  • Автоматическая проверка диапазона
  • Возвращает INT32_MIN если ошибка

  prvGetParamString(cmd, param_num, buffer, size)
  • Получить строку из параметра
  • Безопасное копирование с проверкой размера
  • Возвращает NULL если ошибка

  prvGetParamChar(cmd, param_num)
  • Получить один символ (A-I, и т.д.)
  • Автоматически в верхний регистр
  • Возвращает 0 если ошибка

  prvParamEquals(cmd, param_num, expected)
  • Проверить совпадение параметра со строкой
  • Точное совпадение длины
  • Возвращает pdTRUE/pdFALSE

РЕЗУЛЬТАТ:
  ✓ -50% boilerplate кода в каждой команде
  ✓ Единообразная обработка ошибок
  ✓ Переиспользуются во всех командах
